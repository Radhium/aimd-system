| Matrix multiplication       | Shop bill — customers × quantities paired with prices × 1. Middle number is the shared thing (products). If it doesn't match, a price has nothing to multiply against.                                                                       | Session 2          |
| Chain rule                  | Oven → browning → timing → moisture. Each step only knows its neighbours. Multiply the effects as you walk back through the chain.                                                                                                           | Session 3          |
| self in a class             | A name tag each object wears. `self.count` means "the count that belongs to this specific object." Without it, data evaporates after each function call.                                                                                     | Session 3          |
| Computational graph         | A shop receipt — shows every item and price, not just the total. Backpropagation reads it backwards to find where error came from.                                                                                                           | Session 4          |
| CPU vs GPU                  | Your desk vs a workbench across the room. Faster for big jobs but you have to carry the work over yourself — PyTorch never moves it without being asked.                                                                                     | Session 4          |
| Input vs weight             | The photo vs your intuition in a guessing game. The photo changes every round (input). Your internal mental model improves over time (weight). The weight starts naive — 0.5 is not meaningful, just a starting point that is clearly wrong. | Session 5          |
| Gradient sign and direction | Gradient is negative → subtracting a negative → weight goes up. Direction is not chosen — it falls out of the arithmetic automatically.                                                                                                      | Session 5          |
| Output layer weight         | A volume knob — one per hidden neuron. Near zero means the network has turned that neuron off. Near 1 means it trusts it strongly.                                                                                                           | Session 7          |
| Convergence                 | Sharpening a pencil that is already sharp — more training after convergence produces almost no change.                                                                                                                                       | Session 7          |
| Cross-entropy loss          | A confidence penalty — the more certain you are about the wrong answer, the harder you get punished. Being 99% confident in the wrong class hurts far more than being 50% uncertain.                                                         | Session 8          |
| Token IDs                   | Locker numbers at school — Locker 50 is not "more" than Locker 1. They are labels, not values. No numeric relationship between tokens.                                                                                                       | Session 9          |
| Positional encoding         | Writing a number on the back of each index card before handing the stack to the Transformer — the card itself says what the word is, the number on the back says where it came from.                                                         | Session 9          |
| Self-attention (Q/K/V)      | A library search — Query is what you're looking for, Key is the label on the spine of each book, Value is the content inside. Every token is simultaneously a searcher, a label, and a book.                                                 | Session 10         |
| Residual connection         | Adding a shortcut lane on a motorway — gradients can travel the fast lane directly without going through every junction. Each block only needs to learn the detour, not the whole route.                                                     | Session 10         |
| Attention vs FFN            | Attention = communication (tokens talk to each other). FFN = computation (each token processes what it heard).                                                                                                                               | Session 10         |
| Causal masking (-inf → 0)   | Setting a position to -inf is not deprioritising it — it's making it mathematically cease to exist. e^(-inf) = 0, not "a very small number." The future token is not just ignored, it is gone from the attending token's perspective.        | Session 12         |
| Cosine LR schedule          | Driving a car — early in training you're on a motorway (big steps are fine, get there faster). Late in training you're parking in a tight spot (small careful steps to settle). The cosine curve is the smooth transition between the two.   | Session 14         |
| Scale vs architecture       | Your model and GPT-4 are built from the same blueprints. The difference is GPT-4 is a skyscraper and yours is a garden shed — same design principles, completely different scale of materials and time.                                      | Session 15         |
| Catastrophic forgetting     | A notebook you can only write on — not a notebook you can add pages to. Each new training run overwrites old pages. The previous writing doesn't accumulate: it fades.                                                                       | Session 15         |

| Class                         | A blueprint for building objects. | self                          | A reference an object uses to point back to itself — how it stores and accesses its own data between function calls                                                                                                                                                                   | Session 3                      |

| PyTorch tensor                | A NumPy-like array with two added superpowers: it can live on the GPU, and it can remember every operation performed on it (autograd)                                                                                                                                                 | Session 4                      |
| `requires_grad`               | A flag you set on a tensor to tell PyTorch to watch it and record every operation that touches it                                                                                                                                                                                     | Session 4                      |

| `.backward()`                 | The PyTorch call that reads the computational graph backwards and computes all gradients automatically                                                                                                                                                                                | Session 4                      |
| `.grad`                       | The attribute on a tensor that holds its computed gradient after `.backward()` has been called                                                                                                                                                                                        | Session 4                      |

The original stays where it was — always a copy, not a move.                                                                                                                                                   | Session 4                      |
| `cuda:0`                      | NVIDIA's GPU system, device number zero. The `:0` is an index — if you have one GPU it is always zero. | Forward pass                  | Data moving through the network in one direction — input in, prediction out. No learning happens during this step.                                                                                                                                                                    | Session 5                      |

| MSE (Mean Squared Error)      | A loss function: (prediction − target)². Squaring removes the sign and punishes large errors more than small ones.                                                                                                                                                                    | Session 5                      |

In training, comes from the dataset.                                                                                                                                                                                            | Session 5                      |
| Learning rate                 | A small number that controls how large each weight update step is. Too large → overshoot. Too small → trains very slowly.                                                                                                                                                             | Session 5                      |
| Training step                 | One full cycle: forward pass → loss → backward pass → weight update → zero gradient.                                                                                                                                                                                                  | Session 5                      |
| `weight.grad.zero_()`         | Clears the gradient on a weight before the next training step. | `torch.no_grad()`             | A context block that tells PyTorch not to record the operations inside it. Used during weight updates and validation — we don't want those tracked in the computational graph.                                                                                                        | Session 5                      |
| Matplotlib                    | A Python library for drawing charts. `plt.show()` opens a window with the plot after training finishes. No longer used from Session 8 onward.                                                                                                                                         | Session 5                      |
| Divergence                    | When the learning rate is too large, the weight overshoots the target, overcorrects, and never settles. The loss explodes instead of falling.                                                                                                                                         | Session 5                      |
| Bias (neuron)                 | A learnable number a neuron adds to its weighted sum before the activation — shifts the output up or down regardless of the input                                                                                                                                                     | Session 6                      |
| Activation function           | A function applied after the weighted sum — introduces a bend (non-linearity) so stacking layers adds real expressive power                                                                                                                                                           | Session 6                      |
| ReLU                          | The simplest activation function: max(0, z). Passes positive values unchanged, outputs zero for negatives.                                                                                                                                                                            | Session 6                      |
| `nn.Module`                   | PyTorch's base class for all models — automatically tracks all weights and biases declared inside it                                                                                                                                                                                  | Session 6                      |
| `nn.Linear`                   | A built-in PyTorch layer — creates weights and biases automatically for a given number of inputs and neurons                                                                                                                                                                          | Session 6                      |
| `nn.ReLU`                     | PyTorch's built-in ReLU activation — applied as a layer in the forward pass                                                                                                                                                                                                           | Session 6                      |
| `nn.MSELoss`                  | PyTorch's built-in MSE loss function — same as manual (prediction − target)² but handles batches cleanly                                                                                                                                                                              | Session 6                      |

| Inference                     | Running a trained model on new inputs to get predictions — no training, no gradient tracking needed                                                                                                                                                                                   | Session 6                      |
| Weighted sum (dot product)    | Each input multiplied by its own weight, all results summed together — the core computation inside every neuron                                                                                                                                                                       | Session 7                      |
| Convergence                   | The point during training where loss has bottomed out and further epochs produce no meaningful change                                                                                                                                                                                 | Session 7                      |
| Distributed representation    | The network's solution is spread across many neurons — no single neuron holds the full answer                                                                                                                                                                                         | Session 7                      |
| Logits                        | Raw output scores from the final layer — before Softmax turns them into probabilities. | Softmax                       | Turns raw scores into probabilities: all outputs positive, all sum to 1.0, ranking preserved. Applied internally by `nn.CrossEntropyLoss` during training.                                                                                                                            | Session 8                      |
| Cross-entropy loss            | Loss for classification: `-log(probability of correct class)`. Punishes confident wrong answers hardest.                                                                                                                                                                              | Session 8                      |
| `nn.CrossEntropyLoss`         | PyTorch's built-in classification loss — takes raw logits and integer labels, applies Softmax internally                                                                                                                                                                              | Session 8                      |
| `torch.argmax`                | Returns the index of the highest value in a tensor — used to pick the predicted class from logits                                                                                                                                                                                     | Session 8                      |
| Accuracy                      | Percentage of predictions that match the true label — correct / total                                                                                                                                                                                                                 | Session 8                      |
| Token                         | The smallest unit of text the model processes. In character-level models, one character = one token.                                                                                                                                                                                  | Session 9                      |
| Token ID                      | The integer assigned to a token by the tokeniser. A label, not a numeric value — locker numbers, not measurements.                                                                                                                                                                    | Session 9                      |
| Embedding                     | A learned vector representation of a token. Retrieved by looking up the token's ID in the embedding table.                                                                                                                                                                            | Session 9                      |
| Embedding dimension (d_model) | The length of each token's vector. Every token in the model has a vector of this length.                                                                                                                                                                                              | Session 9                      |
| Embedding table               | A matrix of shape `(vocab_size, embedding_dim)`. One row per token. Learned during training.                                                                                                                                                                                          | Session 9                      |
| Positional encoding           | A second embedding table — shape `(max_seq_len, embedding_dim)` — that encodes position index rather than token identity. Added element-wise to token embeddings.                                                                                                                     | Session 9                      |
| Sequence length               | The number of tokens in the current input. Determines how many rows are retrieved from the embedding and positional encoding tables.                                                                                                                                                  | Session 9                      |
| Query (Q)                     | A vector representing what a token is searching for in the sequence. Produced by multiplying the token's embedding by the learned matrix W_Q.                                                                                                                                         | Session 10                     |
| Key (K)                       | A vector representing what a token contains — its searchable label. Produced by multiplying the token's embedding by W_K.                                                                                                                                                             | Session 10                     |
| Value (V)                     | A vector representing the content a token contributes when attended to. Produced by multiplying the token's embedding by W_V.                                                                                                                                                         | Session 10                     |
| Attention map                 | The `(seq_len, seq_len)` matrix of attention weights — each row shows how much one token attends to every other token.                                                                                                                                                                | Session 10                     |
| Multi-head attention          | Running h independent attention heads in parallel, each learning different token relationships, then concatenating and projecting their outputs back to d_model.                                                                                                                      | Session 10                     |
| W_O                           | The output projection matrix in multi-head attention — maps the concatenated head outputs back to shape `(seq_len, d_model)`.                                                                                                                                                         | Session 10                     |
| Feed-forward layer (FFN)      | Two linear layers with GELU between them inside each Transformer block. Expands to 4×d_model then compresses back. Processes each token independently after attention.                                                                                                                | Session 10                     |
| GELU                          | Smooth activation function used in Transformers. Like ReLU but allows small negative values through — better gradient flow than a hard threshold.                                                                                                                                     | Session 10                     |
| Layer normalisation           | Normalises each token's vector to have stable mean and variance. Applied before attention and before FFN in Pre-LN style. Shape in = shape out.                                                                                                                                       | Session 10                     |
| Residual connection           | Adding the block's input back to its output: `x = x + block(x)`. Requires matching shapes. Stabilises training and gives gradients a direct path backwards.                                                                                                                           | Session 10                     |
| Pre-LN                        | Layer norm applied before each sub-block (attention, FFN) rather than after. Modern standard for GPT-style models.                                                                                                                                                                    | Session 10                     |
| Causal mask                   | A lower-triangular boolean mask applied to attention scores before Softmax. Sets upper-triangle (future) positions to -inf, which Softmax turns to exactly zero. Prevents the model from reading future tokens during training.                                                       | Session 12                     |
| Lower triangular matrix       | A matrix where all values above the diagonal are zero. The shape of the attention weights after causal masking — each token attends only to itself and earlier positions.                                                                                                             | Session 12                     |
| `torch.tril`                  | PyTorch function that returns the lower triangle of a matrix — zeros everything above the diagonal. Used to build the causal mask.                                                                                                                                                    | Session 12                     |
| `masked_fill`                 | PyTorch method that replaces positions where a boolean mask is True with a given value. Used to insert -inf into the upper triangle of attention scores.                                                                                                                              | Session 12                     |
| `model.train()`               | PyTorch call that puts a model into training mode — dropout is active, behaviour is stochastic.                                                                                                                                                                                       | Session 12                     |
| `model.eval()`                | PyTorch call that puts a model into evaluation mode — dropout is off, behaviour is deterministic. Always call before validation.                                                                                                                                                      | Session 12                     |
| Checkpoint                    | A saved copy of model weights written to disk when validation loss reaches a new best. Allows training to be resumed or the best model to be recovered.                                                                                                                               | Session 12                     |
| Encoding (tokenizer)          | Converting text into a list of integer token IDs using char_to_id — feeds text into the model.                                                                                                                                                                                        | Session 12                     |
| Decoding (tokenizer)          | Converting a list of integer token IDs back into readable text using id_to_char — reads what the model generates.                                                                                                                                                                     | Session 12                     |
| Cosine annealing              | A learning rate schedule that starts high and smoothly reduces to near-zero following the shape of a cosine curve. Implemented in PyTorch as `CosineAnnealingLR`. Tested in Experiment #3 — did not improve results for the small model.                                              | Session 14                     |
| Negative result               | An experiment that does not improve on the baseline. Valuable because it rules out a hypothesis and teaches something about the model. Not a failure — a data point. Experiments #3 and #4 (Shakespeare) were both negative results that led directly to the better dataset decision. | Session 14                     |
| Temperature (generation)      | A number that controls how random the model's output is during generation. Low temperature (0.7) = more predictable, picks higher-probability tokens. High temperature = more creative and random. Temperature 0 = always greedy (always picks the top token).                        | Session 14–15                  |
| Catastrophic forgetting       | When a neural network is trained on new data, weights shift toward the new distribution and old learning fades. Networks do not accumulate knowledge the way humans do — new training overwrites old.                                                                                 | Session 15                     |
| Instruction fine-tuning       | A second training phase after pretraining where a model is trained on (question → answer) pairs so it learns to respond to instructions rather than just continue text. The step that turns a raw LM into an assistant. Not possible in this project at current scale.                | Session 15                     |
| Quantisation                  | Compressing model weights to use less memory — allows large models (7B+ parameters) to fit and run on consumer GPU VRAM. Used by Ollama to run models like Llama. Different from training from scratch.                                                                               | Session 15                     |
| Project Gutenberg             | A free, public domain digital library with thousands of books in plain text format. Used as the upgrade dataset for Experiment #4 — 5 books concatenated into 5.5MB of clean literary English prose.                                                                                  | Session 15                     |
